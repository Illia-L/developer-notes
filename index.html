Developing Images

1. Using <picture> by Default

Use <picture> for:

  • all images under 100 KB
  • images 100 - 200 KB, except for hero section, banners and other visual high-priority places 

<picture>
    <source srcset="image.avif" type="image/avif">
    <source srcset="image.webp" type="image/webp">
    <img src="image.jpg" alt="Responsive and optimized image" loading="lazy">
</picture>

2. Placeholder for Heavy Images

Use a placeholder for:

  • images over 200 KB
  • 100 - 200 KB high-priority images, like hero or banner visuals. 

<div class="image-wrapper" style="position: relative;">
    <!-- Placeholder image shown first -->
    <img src="placeholder.jpg" alt="" aria-hidden="true" 
          style="position: absolute; top: 0; left: 0; width: 100%; height: auto;">
    
    <!-- Main image with lazy loading -->
    <img srcset="image-600.jpg 600w, image-1200.jpg 1200w" 
          sizes="(max-width: 600px) 100vw, 600px" 
          src="image-1200.jpg"
          alt="High-priority image with placeholder"
          loading="lazy">
</div>

3. JavaScript Fallback for Broken Images

Use JavaScript fallback for:

  • Ensuring all images have a backup if they fail to load.
  • Applying fallback universally on pages with many dynamic images. 

<script>
document.addEventListener("error", function(e) {
    if (e.target.tagName === "IMG") {
        e.target.src = "fallback.jpg";  // Default fallback image
    }
}, true);
</script>

Clone GitHub repo, set your branch, and make first commit


# 1. Clone the GitHub repository (replace 'URL' with the repo's URL)
git clone https://github.com/username/repository-name.git

# 2. Navigate into the cloned repository
cd repository-name

# 3. Create and switch to a new branch (replace 'branch-name' with your desired branch name)
git checkout -b branch-name

# 4. Make changes to your files (edit files as needed)

# 5. Add the modified files to the staging area
git add .

# 6. Commit the changes with a descriptive message
git commit -m "First commit on branch-name"

# 7. Push the branch to GitHub
git push -u origin branch-name

# 8. That's all, further pushes you can do with simply
git push

Create a GitHub repo from existing project on local machine


# 1. Navigate to your project directory
cd /path/to/your/project

# 2. Initialize a new git repository
git init

# 3. Add all project files to the repository
git add .

# 4. Commit the changes with a descriptive message
git commit -m "Initial commit"

# 5. Create a new repository on GitHub using GitHub CLI (replace 'repository-name' with your desired repo name)
gh repo create repository-name --public --source=. --remote=origin

# 6. Push the local commits to the GitHub repository
git push -u origin main
