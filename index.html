useReducer in React

Follow these steps to use useReducer for state management.

1. Import useReducer and define reducer

Create a reducer function to handle state updates:


import { useReducer } from "react";

function reducer(state, action) {
  switch (action.type) {
      case "increment":
          return { count: state.count + 1 };
      case "decrement":
          return { count: state.count - 1 };
      default:
          return state;
  }
}

2. Initialize useReducer in a component

Use useReducer inside a functional component:


function Counter() {
  const [state, dispatch] = useReducer(reducer, { count: 0 });

  return (
      <div>
          <p>Count: {state.count}</p>
          <button onClick={() => dispatch({ type: "increment" })}>+</button>
          <button onClick={() => dispatch({ type: "decrement" })}>-</button>
      </div>
  );
}

React Router

Use React Router to handle navigation in a React application. Follow these steps to install and use it effectively.

1. Install React Router

To use React Router, install the package:

npm install react-router-dom

2. Basic usage example

Wrap your app in a BrowserRouter to enable routing: 


import { BrowserRouter as Router, Routes, Route } from "react-router-dom";

function App() {
  return (
      <Router>
          <Routes>
              <Route path="/" element={<Home />} />
              <Route path="/about" element={<About />} />
          </Routes>
      </Router>
  );
}

3. Creating links to routes

Use Link instead of <a> to navigate between pages: 


import { Link } from "react-router-dom";

function Navbar() {
  return (
      <nav>
          <Link to="/">Home</Link>
          <Link to="/profile">Profile</Link>
      </nav>
  );
}

4. Redirecting to a route

Use useNavigate to programmatically navigate:


import { useNavigate } from "react-router-dom";

function Login() {
  const navigate = useNavigate();
  
  function handleLogin() {
      // Perform login logic
      navigate("/dashboard"); // Redirect after login
  }

  return <button onClick={handleLogin}>Log In</button>;
}

5. Getting the page from the address bar

Use useLocation to access the current path:


import { useLocation } from "react-router-dom";

function CurrentPage() {
  const location = useLocation();
  
  return <p>Current path: {location.pathname}</p>;
}

// Example URL: https://example.com/profile
// Rendered output: Current path: /profile

6. Working with query parameters

Use useSearchParams to get and set query parameters: 


import { useSearchParams } from "react-router-dom";

function FilterPage() {
  const [searchParams, setSearchParams] = useSearchParams();

  function updateFilter() {
      setSearchParams({ category: "books" }); // Updates URL to ?category=books
  }

  return (
      <div>
          <p>Current category: {searchParams.get("category")}</p>
          <button onClick={updateFilter}>Set Filter</button>
      </div>
  );
}

// Example URL: https://example.com/shop?category=electronics
// Rendered output: Current category: electronics

// After clicking "Set Filter", URL changes to: https://example.com/shop?category=books
// New rendered output: Current category: books

Context API

Use the Context API in React to manage state globally without passing props down manually at every level. Follow these three steps:

1. Create context

Define a new context:

const MyContext = createContext();

2. Create provider

Wrap your app (or a part of it) with the context provider and pass values: 


function MyProvider({ children }) {
  const value = { key: "value" }; // Your shared state or functions
  return (
      <MyContext.Provider value={value}>
          {children}
      </MyContext.Provider>
  );
}

3. Use context

Access the context in a component using useContext: 


function MyComponent() {
  const { key } = useContext(MyContext);
  return ...
}

Get another developer's branch from GitHub

Follow these steps to work with another developer's branch from a GitHub repository. The commands differ based on whether you already have the branch locally or need to fetch it from the remote repository.

If the branch is already on your local machine:

  1. Switch to the branch: 
    git checkout <branch-name>
  2. Pull the latest changes from the remote repository: 
    git pull origin <branch-name>

If the branch is not on your local machine:

  1. Fetch the branch from the remote repository: 
    git fetch origin <branch-name>
  2. Create and switch to the branch locally: 
    git checkout -b <branch-name> origin/<branch-name>

Roll back remote main branch to a specific commit

  1. View the commit history to identify the target commit hash: 
    git log --format="%H %s" --all
  2. Reset your local main branch to the desired commit (replace <commit-hash> with the actual hash): 
    git reset --hard <commit-hash>
  3. Force-push the rolled-back main branch to the remote repository: 
    git push origin main -f

Note: The -f (force) option on git push overwrites the remote branch history. Ensure that this change is communicated to any collaborators who may have pulled newer commits, as they will need to reset their local branches to match the updated remote main.

Merge Another Developer's Branch to the Main


# 1. Ensure you're on the main branch
git checkout main

# 2. Fetch all branches from the remote repository
git fetch origin

# 3. Merge the other developer's branch directly from the remote
# Replace "other-branch" with the actual branch name you want to merge
git merge origin/other-branch

# 4. Push the changes to the remote main branch if needed
git push origin main

Developing Images

1. Using <picture> by Default

Use <picture> for:

  • all images under 100 KB
  • images 100 - 200 KB, except for hero section, banners and other visual high-priority places 

<picture>
    <source srcset="image.avif" type="image/avif">
    <source srcset="image.webp" type="image/webp">
    <img src="image.jpg" alt="Responsive and optimized image" loading="lazy">
</picture>

2. Placeholder for Heavy Images

Use a placeholder for:

  • images over 200 KB
  • 100 - 200 KB high-priority images, like hero or banner visuals. 

<div class="image-wrapper" style="position: relative;">
    <!-- Placeholder image shown first -->
    <img src="placeholder.jpg" alt="" aria-hidden="true" 
          style="position: absolute; top: 0; left: 0; width: 100%; height: auto;">
    
    <!-- Main image with lazy loading -->
    <img srcset="image-600.jpg 600w, image-1200.jpg 1200w" 
          sizes="(max-width: 600px) 100vw, 600px" 
          src="image-1200.jpg"
          alt="High-priority image with placeholder"
          loading="lazy">
</div>

3. JavaScript Fallback for Broken Images

Use JavaScript fallback for:

  • Ensuring all images have a backup if they fail to load.
  • Applying fallback universally on pages with many dynamic images. 

<script>
document.addEventListener("error", function(e) {
    if (e.target.tagName === "IMG") {
        e.target.src = "fallback.jpg";  // Default fallback image
    }
}, true);
</script>

Clone GitHub repo, set your branch, and make first commit


# 1. Clone the GitHub repository (replace 'URL' with the repo's URL)
git clone https://github.com/username/repository-name.git

# 2. Navigate into the cloned repository
cd repository-name

# 3. Create and switch to a new branch (replace 'branch-name' with your desired branch name)
git checkout -b branch-name

# 4. Make changes to your files (edit files as needed)

# 5. Add the modified files to the staging area
git add .

# 6. Commit the changes with a descriptive message
git commit -m "First commit on branch-name"

# 7. Push the branch to GitHub
git push -u origin branch-name

# 8. That's all, further pushes you can do with simply
git push

Create a GitHub repo from existing project on local machine


# 1. Navigate to your project directory
cd /path/to/your/project

# 2. Initialize a new git repository
git init

# 3. Add all project files to the repository
git add .

# 4. Commit the changes with a descriptive message
git commit -m "Initial commit"

# 5. Create a new repository on GitHub using GitHub CLI (replace 'repository-name' with your desired repo name)
gh repo create repository-name --public --source=. --remote=origin

# 6. Push the local commits to the GitHub repository
git push -u origin main
